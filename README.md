# 二手货品置换平台

一个基于 **Go 后端 + 原生 HTML/JS** 的二手货品置换平台业务骨架。
Go 服务同时提供 API 接口和页面静态资源，**浏览器打开 `http://localhost:8080` 直接进入工作台**，无需额外前端端口。

---

## 📁 项目结构

```
.
├── cmd/
│   └── server/
│       └── main.go              # 服务入口，路由注册
├── internal/
│   ├── model/
│   │   └── item.go              # 数据模型（货品、请求、统计）
│   ├── data/
│   │   └── mock_data.go         # 8条本地模拟数据
│   ├── service/
│   │   └── item_service.go      # 业务逻辑层（CRUD、操作、统计）
│   └── handler/
│       └── item_handler.go      # HTTP 接口处理器
├── web/
│   ├── static/
│   │   ├── css/style.css        # 页面样式
│   │   └── js/app.js            # 前端交互逻辑
│   └── templates/
│       └── index.html           # 工作台页面（单页）
├── go.mod
└── README.md
```

## 🛠 技术栈

- **后端**: Go 1.21+ 标准库 `net/http`（零第三方依赖）
- **前端**: 原生 HTML5 + CSS3 + JavaScript (ES6+)
- **数据**: 本地内存模拟（`sync.RWMutex` 并发安全）
- **服务收口**: Go 服务统一提供页面、静态资源和 API

## 🔌 API 接口列表（全部通过 `http://localhost:8080` 访问）

| 方法 | 路径 | 说明 |
|------|------|------|
| **GET** | `/` | **工作台首页入口**（直接返回 HTML） |
| GET | `/static/*` | 静态资源（CSS/JS） |
| GET | `/api/items` | 获取货品列表（支持 `keyword`/`category`/`city`/`status` 查询参数） |
| GET | `/api/items/{id}` | 获取货品详情（自动 +1 浏览数） |
| POST | `/api/items` | 发布新货品 |
| PUT | `/api/items/{id}` | 编辑货品 |
| DELETE | `/api/items/{id}` | 删除货品 |
| GET | `/api/statistics` | 获取统计数据 |
| GET | `/api/meta` | 获取元数据枚举（品类/城市/成色/状态） |
| POST | `/api/items/{id}/favorite` | 收藏/取消收藏 |
| POST | `/api/items/{id}/trade-intent` | 发起置换意向（**已下架货品返回 400 错误，不更新数据**） |
| POST | `/api/items/{id}/mark-communicated` | 标记已沟通 |
| POST | `/api/items/{id}/offline` | 下架 |
| POST | `/api/items/{id}/relist` | 重新上架 |

所有接口统一响应格式：`{ code: number, message: string, data: any }`
- `code < 400`：成功
- `code >= 400`：失败，`message` 包含错误信息

### 5. 详情操作闭环（5 个核心按钮）

| 按钮 | 操作 | 预期结果 |
|------|------|----------|
| ⭐ **收藏/取消收藏** | 点击 | 按钮变黄色，收藏数 ±1，列表和统计同步更新 |
| 💬 **发起置换意向** | 点击（上架中货品） | 成功 Toast，意向数 +1 |
| 💬 **发起置换意向** | 点击（已下架货品） | ⚠️ **橙色警告 Toast**，按钮灰，**意向数不变，统计不变** |
| 📞 **标记已沟通** | 点击 | 按钮变蓝色，状态改为 ✓ 已沟通 |
| ⬇️ **下架** | 点击确认 | 状态变「已下架」，下架按钮消失，出现「重新上架」按钮 |
| ⬆️ **重新上架** | 点击（已下架货品） | 状态恢复「上架中」，重新显示「下架」按钮 |
| ✏️ **编辑** | 点击 → 修改 → 提交 | 列表和详情内容同步更新，发布者字段不可编辑 |

### 6. 发布新货品
- ✅ 点击右上角「+ 发布货品」→ 弹出表单
- ✅ 必填字段：标题、品类、城市、成色、发布者、期望置换品类
- ✅ 提交后：列表顶部出现新货品，自动选中显示详情，统计「总货品」+1

### 7. 已下架货品异常拦截（关键复查项）
**操作步骤**：
1. 选择 ID=4「宜家三人布艺沙发」（初始状态就是已下架）
2. 观察：「发起置换意向」按钮呈灰色 `disabled` 状态
3. 即使通过其他方式触发：后端返回 400，前端 catch 后显示 **「⚠️ 该货品已下架，无法发起置换意向」**
4. ✅ **重要**：货品意向数、列表显示、顶部统计卡片 **全部不发生任何变化**

## 🧱 分层架构说明（便于后续扩展）

| 层级 | 文件 | 职责 |
|------|------|------|
| **入口层** | `cmd/server/main.go` | 路由注册、服务启动 |
| **处理层** | `internal/handler/item_handler.go` | HTTP 请求解析、响应封装 |
| **服务层** | `internal/service/item_service.go` | 业务规则、状态校验、数据修改 |
| **数据层** | `internal/data/mock_data.go` | 本地模拟数据（后续替换为数据库 DAO） |
| **模型层** | `internal/model/item.go` | 数据结构定义（请求/响应/实体） |
| **表现层** | `web/static/js/app.js` | 页面渲染、事件绑定、状态管理 |

各层之间通过明确的接口和数据结构解耦，后续可独立扩展：
- **角色权限**：在 handler 层增加中间件，service 层增加权限校验
- **异常状态**：在 model 的 `ItemStatus` 增加枚举，service 层增加状态流转规则
- **持久化**：将 service 层的内存 map 替换为数据库操作，接口签名不变

# 二手货品置换平台

一个基于 Go + 原生 HTML/JS 的二手货品置换平台业务骨架。

## 项目结构

```
.
├── cmd/
│   └── server/
│       └── main.go              # 服务入口
├── internal/
│   ├── model/
│   │   └── item.go              # 数据模型定义
│   ├── data/
│   │   └── mock_data.go         # 本地模拟数据
│   ├── service/
│   │   └── item_service.go      # 业务逻辑服务层
│   └── handler/
│       └── item_handler.go      # HTTP 处理器
├── web/
│   ├── static/
│   │   ├── css/
│   │   │   └── style.css        # 页面样式
│   │   └── js/
│   │       └── app.js           # 前端交互逻辑
│   └── templates/
│       └── index.html           # 工作台页面
├── go.mod
└── README.md
```

## 技术栈

- **后端**: Go 1.21+ 标准库 (net/http)
- **前端**: 原生 HTML5 + CSS3 + JavaScript (ES6+)
- **数据**: 本地内存模拟数据，并发安全

## 核心功能

### 页面结构（工作台首页）
- **统计概览区**: 总货品数、上架中、已下架、已置换、总浏览、总收藏、总置换意向
- **货品列表区**: 关键词搜索、品类/城市/状态筛选、列表展示
- **货品详情区**: 完整信息展示、核心操作按钮
- **发布/编辑弹窗**: 货品录入表单

### 货品数据字段
- 标题、品类、城市、成色、发布者、期望置换品类、详细描述
- 浏览数、收藏数、置换意向数
- 状态（上架中/已下架/已置换/待审核）
- 是否已收藏、是否已沟通
- 创建时间、更新时间

### 支持的操作
1. **搜索筛选**: 关键词搜索（标题/描述/发布者）、按品类/城市/状态筛选
2. **发布货品**: 填写表单发布新货品
3. **编辑货品**: 修改已有货品信息
4. **收藏/取消收藏**: 切换收藏状态
5. **发起置换意向**: 对上架中货品发起意向，已下架货品会阻止并提示
6. **标记已沟通**: 标记已与发布者沟通过
7. **下架**: 将货品从上架状态改为下架
8. **重新上架**: 将已下架货品重新上架

### 数据联动特性
- 所有操作后，列表、详情、统计数据会同步自动更新
- 点击列表项自动加载详情并增加浏览数
- 操作结果实时反馈（Toast 提示）

## 本地运行

### 前置条件
- 安装 Go 1.21 或更高版本：https://golang.org/dl/

### 启动步骤
1. 进入项目根目录：
```bash
cd permission-system-f9090ca9-p3
```

2. 运行服务：
```bash
go run cmd/server/main.go
```

3. 打开浏览器访问：
```
http://localhost:8080
```

## API 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 工作台页面 |
| GET | `/api/items` | 获取货品列表（支持 keyword/category/city/status 查询参数） |
| GET | `/api/items/{id}` | 获取货品详情（自动增加浏览数） |
| POST | `/api/items` | 创建新货品 |
| PUT | `/api/items/{id}` | 更新货品信息 |
| DELETE | `/api/items/{id}` | 删除货品 |
| GET | `/api/statistics` | 获取统计数据 |
| GET | `/api/meta` | 获取元数据（品类、城市、成色、状态枚举） |
| POST | `/api/items/{id}/favorite` | 切换收藏状态 |
| POST | `/api/items/{id}/trade-intent` | 发起置换意向 |
| POST | `/api/items/{id}/mark-communicated` | 标记已沟通 |
| POST | `/api/items/{id}/offline` | 下架货品 |
| POST | `/api/items/{id}/relist` | 重新上架 |

- ✅ 城市选择 "北京"，列表只显示北京的货品

### 3. 详情联动验证
- ✅ 点击不同列表项，右侧详情自动切换
- ✅ 每次点击列表项，统计区的 "总浏览" 数 +1
- ✅ 列表当前项有紫色高亮边框

### 4. 核心操作验证
- ✅ 点击 **"☆ 收藏"** 按钮：按钮变为黄色 "★ 已收藏"，列表和统计的收藏数 +1

- ✅ 提交后，列表顶部出现新货品，自动选中并显示详情
- ✅ 统计区总货品数 +1

## 扩展建议

当前架构已为后续扩展预留了清晰的分层结构：

1. **角色权限扩展**:
   - 在 `internal/model` 增加 `User`、`Role`、`Permission` 模型
   - 在 `internal/service` 增加权限校验逻辑
   - 在 `internal/handler` 增加中间件进行权限检查

2. **持久化存储**:
   - 将 `internal/service/item_service.go` 中的内存存储替换为数据库操作
   - 可选择 MySQL、PostgreSQL 或 SQLite
   - 接口签名保持不变，实现可无缝替换

3. **异常状态扩展**:
   - 在 `internal/model/item.go` 的 `ItemStatus` 增加更多状态
   - 在 `internal/service` 增加状态流转的业务规则校验

4. **前端增强**:
   - 可引入 Vue/React 等框架替换原生 JS
   - 增加用户登录注册页面
   - 增加消息通知系统
